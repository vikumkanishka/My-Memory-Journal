UI explanation

Memory Journal - labeled UI guide

This document maps the primary screens in the application and names each visible section so any viewer can quickly understand how the interface is organized.

Format Desktop browser mockups with callout labels and arrows.
Style Clean SaaS layout with soft surfaces, rounded cards, and light theme styling.
Naming Consistent terms across screens: Navbar, Page Header, Main Content Area, and more.

The screens below are documentation illustrations, not interactive app pages.

1. Home Dashboard

Shows the landing experience of index.html with the main navigation, welcome banner, quick actions, recent entries, tracker shortcuts, and profile access.

Navbar: Primary navigation, logo, and profile access.
Welcome Section: Main hero area introducing the dashboard.
Tracker Shortcuts: Quick entry points to the habit modules.

2. Journal Editor Page

Shows new.html with the writing workspace, toolbar, metadata fields, mood selector, image tools, and action buttons.

Editor Area: Primary writing surface where the entry content is composed.
Tags Input: Metadata field for organizing entries.
Image Upload Area: Space for attaching photos to a memory.

3. Entries List Page

Shows entries.html with the search and filter bar, grid or list toggle, entry cards, and pagination controls.

Entry Card: One memory preview with metadata and quick access actions.
Filter Controls: Sort, mood, and layout controls that narrow results.
Pagination / Load More: Navigation for longer entry histories.

4. Single Entry View Page

Shows view.html as a focused reading layout with the title, content, images, tags, and edit/delete actions.

Entry Content: Full article-style reading area for the selected journal post.
Images Section: Visual attachments that support the written memory.
Action Buttons: Controls for editing or removing the entry.

5. Profile Settings Page

Shows edit-profile.html with profile information, input fields, theme controls, and the save action.

Profile Information: Section for identity and contact details.
Theme Settings: Appearance controls for the light or dark mode selection.
Save Changes: Primary action that stores the updates locally.

6. Tracker Page

Shows the shared layout used by the habit trackers such as mood, water, gym, study, period, expense, and gratitude.

Input Section: Logging controls for the selected tracker.
Stats Summary: Totals, streaks, averages, and other key measurements.
History List: Timeline of previously recorded tracker data.

Naming standard used across the guide: Navbar, Page Header, Main Content Area, Sidebar Navigation, Stats Summary, Entry Card, and Action Buttons.
